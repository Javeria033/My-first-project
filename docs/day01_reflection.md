# Day 1 Reflection

## What was the most difficult part of the activity?
The most challenging part was setting up the virtual environment and ensuring the project runs in a clean environment without errors. Understanding Git branching and creating a proper pull request also required careful attention.

## What did you learn about reproducibility?
Reproducibility means that anyone can clone the repository, install dependencies, and run the project without additional setup. Using a `requirements.txt` file and virtual environments ensures consistency across different machines.

## Why are virtual environments important?
Virtual environments isolate project dependencies from the global Python installation. This prevents version conflicts between different projects and ensures consistent behavior.

## What makes a Git commit professional?
A professional Git commit has a clear, descriptive message that explains what changed and why. It follows conventions like using prefixes (feat:, fix:, docs:, chore:) and keeps messages concise but informative.

## What is the purpose of a pull request?
A pull request allows team members to review code changes before merging them into the main branch. It enables collaboration, catches bugs early, and maintains code quality.

## What would you improve in your repository?
I would add more comprehensive test cases, include type hints for all functions, and set up GitHub Actions for automated testing. Adding visualizations would also enhance the project.

## Which AI team role interests you most, and why?
The Data Scientist role interests me the most because it combines statistical analysis, programming, and problem-solving. I enjoy finding patterns in data and building models that drive decisions.
